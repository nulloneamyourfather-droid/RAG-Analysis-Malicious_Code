# 基于 RAG 与 IDA-MCP 的恶意代码静态分析技术路线与系统设计

## 1. 课题目标

本课题面向恶意代码静态分析场景，设计并实现一个结合 `IDA-MCP`、`RAG` 与 `MITRE ATT&CK` 知识库的增强分析系统。系统目标不是替代传统逆向分析，而是通过外部知识增强与大模型推理，提高函数级恶意行为识别、ATT&CK 技术映射与样本级报告生成的效率与可解释性。

本课题重点解决以下问题：

1. 如何从 IDA 中稳定提取可用于检索与推理的函数级静态特征。
2. 如何构建适合恶意代码分析的多源知识库，而不是通用文本知识库。
3. 如何将检索结果与当前样本事实结合，形成可解释的 ATT&CK 技术映射结果。
4. 如何将函数级分析结果汇总为样本级恶意行为结论与分析报告。


## 2. 总体技术路线

系统总体技术路线可分为四层：

1. 静态事实采集层
   依托 `IDA-MCP` 从当前 IDA 数据库中提取函数、伪代码、API 调用、字符串、交叉引用、导入表、类型信息等静态事实。

2. 领域知识构建层
   构建面向恶意代码分析的多源知识库，主要包括：
   - `MITRE ATT&CK` 技术知识库
   - `API 行为链` 知识库
   - `IDA 函数语义` 知识库
   - `历史分析报告/经验` 知识库

3. 检索增强推理层
   针对当前函数或样本，先检索相关知识，再结合当前静态事实进行融合推理，输出函数语义、疑似恶意行为和 ATT&CK 映射。

4. 结果汇总与回写层
   将函数级分析结果汇总为样本级行为结论，并在需要时通过 IDA-MCP 将注释、命名、标签或报告回写到 IDA 数据库中。


## 3. 系统总体架构

```text
恶意样本
  ->
IDA Pro / IDA-MCP
  ->
静态特征提取
  - 函数名 / 地址
  - 伪代码 / 反汇编
  - API 调用链
  - 字符串 / 导入表
  - xrefs / 调用关系
  - 类型 / 栈帧 / 注释
  ->
函数级语义表示
  ->
RAG 检索
  - ATT&CK 知识库
  - API 行为链知识库
  - 历史函数语义库
  - 历史报告知识库
  ->
LLM 融合推理
  - 函数行为解释
  - 恶意意图判断
  - ATT&CK technique/tactic 映射
  - 证据链说明
  ->
样本级聚合
  - 多函数行为归并
  - 恶意能力画像
  - 分析报告生成
  ->
结果输出 / 回写 IDA
```


## 4. 系统模块设计

### 4.1 IDA 静态特征提取模块

该模块基于 `IDA-MCP` 获取当前样本的静态事实，重点抽取函数级结构化特征：

- 基础属性：函数名、地址、所属分段、函数大小
- 语义信息：伪代码、反汇编摘要、局部变量、参数信息
- 行为特征：API 调用、导入函数、字符串常量
- 关系特征：调用关系、xrefs、调用者与被调用者
- 补充信息：注释、类型、结构体访问、关键常量

该模块输出的是当前样本的“事实层”，属于高可信输入，不应由检索结果替代。


### 4.2 知识库构建模块

本系统构建四类知识：

#### 4.2.1 ATT&CK 技术知识库

来源于 MITRE ATT&CK STIX 数据，主要包含：

- technique/sub-technique 编号与名称
- tactic 信息
- 平台信息
- 检测思路
- 缓解措施
- 官方描述

该知识库主要用于标准化映射与术语解释。


#### 4.2.2 API 行为链知识库

围绕常见恶意行为建立 API 序列模板，例如：

- 进程注入
- 注册表持久化
- 下载执行
- C2 通信
- 键盘记录
- 环境探测
- 文件批量处理 / 加密

该知识库用于将函数调用链与恶意行为语义建立直接关联，并进一步映射到 ATT&CK technique。


#### 4.2.3 IDA 函数语义知识库

从历史样本中提取函数级语义记录，用于相似函数检索。核心内容包括：

- 样本标识
- 函数地址与名称
- API 调用集合
- 字符串与关键常量
- 函数摘要
- 行为标签
- 伪代码片段

该知识库用于跨样本复用已有分析经验。


#### 4.2.4 历史报告与分析记忆知识库

收录以往样本分析报告、家族总结、函数注释沉淀、经验规则等，用于支持更强的解释性与经验复用。


### 4.3 检索增强模块

检索模块以当前函数的结构化表示为输入，构造检索查询并对多知识库执行召回。可采用双通道召回方式：

1. 稠密检索
   通过 embedding 检索语义相近的函数、技术描述和行为模板。

2. 结构化过滤
   根据平台、API 名、ATT&CK 编号、行为标签、导入函数等元数据进行精确筛选。

最终将多路召回结果合并后送入推理模块。


### 4.4 大模型融合推理模块

该模块接收两类输入：

1. 当前样本事实
   来自 IDA-MCP 的函数级静态特征。

2. 检索增强知识
   来自 ATT&CK、API 行为链、历史函数语义和历史报告的相关片段。

推理目标包括：

- 判断函数的主要行为语义
- 识别可疑恶意能力
- 给出 ATT&CK technique / tactic 映射
- 说明判断依据与证据来源
- 形成函数级分析结论

为控制幻觉，推理输出必须区分：

- 当前样本事实
- 外部知识支持
- 模型推断结论


### 4.5 样本级聚合与报告生成模块

函数级分析完成后，系统将多函数结果聚合为样本级行为画像，例如：

- 是否存在进程注入
- 是否存在持久化
- 是否存在下载执行
- 是否存在 C2 通信
- 是否存在信息窃取
- 是否存在勒索类文件处理行为

最终输出：

- 样本级 ATT&CK 技术集合
- 行为证据链
- 高风险函数列表
- 分析结论与建议


## 5. 数据流设计

### 5.1 离线知识构建流程

```text
ATT&CK STIX / 行为模板 / 历史样本
  ->
清洗与结构化
  ->
统一知识文档格式
  ->
向量化
  ->
写入向量库 + 元数据存储
```


### 5.2 在线分析流程

```text
待分析样本
  ->
IDA-MCP 提取函数级静态特征
  ->
构造检索查询
  ->
检索 ATT&CK / API 行为链 / 历史语义知识
  ->
LLM 融合推理
  ->
生成函数级结论
  ->
聚合为样本级报告
```


## 6. 关键设计原则

### 6.1 当前样本事实优先

ATT&CK 和历史知识只用于增强解释与召回，不能替代当前样本事实。对于函数是否调用某 API、是否存在某字符串、是否有特定 xref，必须以 IDA-MCP 返回结果为准。


### 6.2 多源知识分层使用

不同知识源承担不同职责：

- `ATT&CK`：标准化标签与技术解释
- `API_CHAIN`：行为模式判断
- `IDA_SEMANTIC`：相似函数与历史经验复用
- `REPORT_MEMORY`：家族经验与人工分析沉淀


### 6.3 检索结果必须带元数据

每条检索文档都应包含清晰元数据，至少包括：

- `doc_id`
- `source`
- `doc_type`
- `title`
- `tags`
- `metadata`
- `content`

这样便于后续过滤、重排、消融实验和误差分析。


### 6.4 输出结果需要可解释

系统输出不能只给结论，必须给出：

- 命中的 API / 字符串 / 行为证据
- 对应的 ATT&CK technique
- 检索到的知识来源
- 结论置信说明


## 7. 论文中的实验设计建议

### 7.1 实验目标

验证 RAG 是否能够提升恶意代码静态分析中的：

- 行为识别准确率
- ATT&CK 映射准确率
- 分析解释完整度
- 分析效率


### 7.2 对比组建议

可以设置以下对比：

1. 无知识增强，仅使用 LLM + 当前函数伪代码
2. 使用 ATT&CK 知识增强
3. 使用 ATT&CK + API 行为链增强
4. 使用 ATT&CK + API 行为链 + 历史语义库增强


### 7.3 评价指标建议

- 函数级行为识别准确率
- ATT&CK technique 映射命中率
- Top-k 检索命中率
- 报告解释完整度
- 人工评估满意度


## 8. 本课题的预期成果

本课题预计形成以下成果：

1. 一个基于 `IDA-MCP + RAG + ATT&CK` 的恶意代码静态分析原型系统。
2. 一套面向恶意代码分析的多源知识构建方法。
3. 一套函数级到样本级的行为语义与 ATT&CK 映射流程。
4. 一份可解释的分析报告生成机制。


## 9. 结论

本课题的核心不在于单纯将大模型接入逆向工具，而在于构建“静态事实采集 + 多源知识检索 + 语义推理解释”的完整闭环。通过将 `IDA-MCP` 提供的函数级静态事实与 `MITRE ATT&CK` 及恶意行为模板知识结合，可以显著增强恶意代码静态分析的标准化、解释性与复用能力，为后续恶意代码分析自动化研究提供可实施的技术路径。
